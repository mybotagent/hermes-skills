# Channel Context Discipline

> **한 줄 원칙:** *지금 받은 메시지의 출처* 가 *질문의 답* 이다. 다른 세션은 증거가 아니라 후보일 뿐이다.

이 스킬은 채널/스레드 메시지를 받을 때 반드시 따라야 할 **읽기·응답·검색 행동 규칙**이다. 특정 주제(설문, 포트폴리오 등)에 종속되지 않으며 모든 스레드에 일관 적용된다.

---

## 1. Source of Truth (출처 계층)

계층 순서 — **위가 항상 아래를 이긴다**:

| 순위 | 출처                              | 신뢰도 | 사용 시점                    |
| :--: | :-------------------------------- | :----: | :--------------------------- |
|  1   | `SessionContext.source.*`         |  최상  | 이번 메시지의 진짜 출처      |
|  2   | `system prompt` 의 스레드 메모    |  상    | 이 스레드의 고정 규칙/역할   |
|  3   | `skill` 이 명시한 주제 한정        |  상    | 스킬이 "이 채널 전용"인 경우 |
|  4   | `session_search()` 결과           |  참고  | 참고만, 단정 금지            |
|  5   | `memory`                          |  참고  | 환경/선호도만, 작업맥락X     |

> ⚠️ 4·5번은 "**틀렸을 수 있는 단서**"이지 답이 아니다.

---

## 2. 출처 읽기 단계 (모든 응답 직전)

메시지를 받으면 **반드시** 다음 순서로 출처를 확정한다:

```python
def read_source(context):
    src = context.source             # SessionSource
    return {
        "platform":   src.platform,          # discord, telegram, ...
        "chat_name":  src.chat_name,         # 채널/그룹 이름
        "thread_id":  src.thread_id,         # None 이면 메인 채팅
        "parent_chat_id": src.parent_chat_id,
        "thread_label": src.thread_name
                       or src.chat_topic
                       or src.chat_name,
    }
```

이 값으로 다음을 결정한다:
- **주제 가드** — 스레드 라벨/메모가 명시한 주제가 있으면 그 외 질문은 거절/이관
- **톤 가드** — DM/그룹/스레드별 기본 톤
- **검색 가드** — 4번 출처(session_search)는 1·2·3번과 **모순**될 때 단정 금지

---

## 3. 금지 행동 (Hard NO)

다음 중 하나라도 해당하면 **즉시 멈추고 출처를 다시 본다**:

| # | 금지 행동                                                   | 이유                                |
|:-:|------------------------------------------------------------|-------------------------------------|
| 1 | `session_search()` 결과를 **그대로 인용**해 답하기           | 다른 스레드/세션일 수 있음          |
| 2 | "어제/이전 대화에서~" 로 시작하며 과거 세션을 단정 인용      | 같은 채널 안의 다른 주제일 수 있음  |
| 3 | 스레드 라벨을 무시하고 가장 최근 세션을 "이어서" 가정       | 컨텍스트 오염                       |
| 4 | 메모리 환경설정을 **현재 작업 컨텍스트**처럼 인용            | 메모리는 환경/선호, 작업은 1~3순위  |
| 5 | `home_channels` 가 아닌 채널에서 받은 메시지에 home 응답 가정 | origin = 받은 채널, home ≠ 기본     |

---

## 4. 권장 행동

| # | 행동                                                          | 효과                            |
|:-:|---------------------------------------------------------------|---------------------------------|
| 1 | 첫 응답 1줄에 **현재 출처** 명시  ("`#채널/스레드 에서~`")     | 사용자가 오염 즉시 인지         |
| 2 | `session_search()` 결과는 **가능성** 으로만 제시, 단정 금지    | 추측의 위험을 사용자에게 넘김   |
| 3 | 스레드 라벨과 모순되는 단어는 거절 후 라벨 읽어달라고 요청    | 스레드 경계 회복                |
| 4 | 출처 불명확 시 `clarify` 로 주제 확인                          | 잘못된 답 방지                  |

응답 시작 템플릿 예시:
> `#project-manage / 하루체크리스트 (thread 1520255092413038732) 에서 —`

---

## 5. 자기 검증 체크리스트 (응답 직전)

응답을 보내기 전, 다음에 모두 "예"여야 한다:

- [ ] 현재 메시지의 출처(스레드/채널)를 1줄로 명시했는가?
- [ ] `session_search()` 결과를 단정 인용하지 않았는가?
- [ ] 스레드 라벨/메모가 명시한 주제에서 벗어났는가? (벗어났다면 거절했는가?)
- [ ] 메모리 환경설정을 작업 컨텍스트로 오용하지 않았는가?
- [ ] 응답 채널 = 수신 채널인가? (`home_channels` X)

하나라도 "아니오"면 수정 후 전송.


## 7. 메타

- **범위**: 모든 플랫폼(Discord/Telegram/Slack/Matrix/...) 모든 스레드
- **자동 주입**: system prompt 항상 (system_always: true)
- **충돌 정책**: 다른 주제 한정 스킬(daily-survey 등)이 더 구체적이면 그쪽 우선, 본 스킬은 **상위 가드**
- **테스트 방법**: 가짜 출처로 시나리오 던져 5번 체크리스트 통과 확인
