# Achievements Plugin — Filesystem Data Access

When the Hermes dashboard is behind auth (nginx auth_basic, API server restricted),
or you need programmatic access to achievement progress, go directly to the
plugin's filesystem data.

## File Locations

All paths resolve under `~/.hermes/plugins/hermes-achievements/`.

| File | Purpose |
|------|---------|
| `state.json` | Permanent unlock state — which achievements have been unlocked, at what tier, and when. |
| `scan_snapshot.json` | Current progress snapshot — per-achievement progress, thresholds, tier info. Generated by `hermes insights` rescan. TTL: 120s, refreshed on demand. |
| `scan_checkpoint.json` | Intermediate scan state — raw per-session aggregated metrics used during the scan. Schema-versioned. |

## state.json Schema

```json
{
  "unlocks": {
    "<achievement_id>": {
      "evidence": null,
      "first_tier": "Copper | Silver | Gold | Diamond | Olympian",
      "unlocked_at": <unix_timestamp>
    }
  }
}
```

- `first_tier`: the highest tier achieved at unlock time. `null` for
  multi-condition achievements (e.g. Dependency Hell Tourist) that use `"tier": None`.
- Entries only exist for **unlocked** achievements. Missing ID = locked.
- Unlocks are **immutable** — once saved, they never change.

## scan_snapshot.json Schema

Generated by the achievements engine. Fields per achievement:

| Field | Type | Meaning |
|-------|------|---------|
| `id` | string | Achievement ID (matches state.json keys) |
| `name` | string | Display name |
| `category` | string | Category label |
| `kind` | string | `lifetime`, `best_session`, or `multi_condition` |
| `state` | string | `"unlocked"`, `"discovered"`, or `"secret"` |
| `unlocked` | bool | Whether tier thresholds are met |
| `discovered` | bool | Whether progress > 0 has been observed |
| `secret` | bool | Hidden until progress > 0 |
| `progress` | number | Current metric value |
| `progress_pct` | number | 0.0–100.0 — how close to `next_threshold` |
| `tier` | string|null | Current tier name, or null for multi-condition |
| `next_tier` | string | Next tier name to reach |
| `next_threshold` | number | Value needed for next tier |

**Important caveat**: `progress_pct` can be 0% even when `unlocked=true`
(e.g. File Archaeologist, Skill Issue? Skill Created) — this happens when
the snapshot was generated after the unlock but before the specific metric
accumulated past the first threshold. Always check `unlocked` first.

## Query Patterns

### 1. Get overview (all achievements, sorted by progress)

```bash
cat ~/.hermes/plugins/hermes-achievements/scan_snapshot.json \
  | python3 -c "
import json, sys
d = json.load(sys.stdin)
for a in sorted(d['achievements'], key=lambda x: x['progress_pct'], reverse=True):
    print(f\"  {a['progress_pct']:5.0f}% | {a['name']:40s} | {a['state']:10s} | {a['tier'] or '--':10s} | {a['progress']:>6}/{a['next_threshold']:<6}\")
"
```

### 2. Find near-complete achievements (quick wins)

```bash
cat ~/.hermes/plugins/hermes-achievements/scan_snapshot.json \
  | python3 -c "
import json, sys
d = json.load(sys.stdin)
for a in d['achievements']:
    if 85 <= a['progress_pct'] < 100 and not a['unlocked']:
        print(f\"{a['name']:40s} {a['progress_pct']:.0f}% — {a['progress']}/{a['next_threshold']} to {a['next_tier']}\")
"
```

### 3. Find 0% / untouched areas (diversity gaps)

```bash
cat ~/.hermes/plugins/hermes-achievements/scan_snapshot.json \
  | python3 -c "
import json, sys
d = json.load(sys.stdin)
for a in d['achievements']:
    if a['progress'] <= 1 and not a['unlocked'] and a['state'] != 'secret':
        print(f\"{a['name']:40s} ({a['id']}) — {a['progress']}/{a['next_threshold']}\")
"
```

### 4. Count unlocks by category

```bash
cat ~/.hermes/plugins/hermes-achievements/state.json \
  | python3 -c "
import json, sys
d = json.load(sys.stdin)
print(f\"Total unlocked: {len(d['unlocks'])}\")
for k, v in sorted(d['unlocks'].items()):
    print(f\"  {k:40s} tier={v.get('first_tier', '?')}\")
"
```

### 5. Trigger a rescan

Access the dashboard API or restart the gateway — the snapshot auto-refreshes
via the `/api/plugins/hermes-achievements/overview` endpoint. There's no CLI
command to trigger a rescan; use `hermes insights --days N` to force a re-read.

## Dashboard API (alternative)

When accessible without auth:

```
GET /api/plugins/hermes-achievements/overview
GET /api/plugins/hermes-achievements/scan_status
```

Returns the same data as `scan_snapshot.json` plus scan-status metadata.
