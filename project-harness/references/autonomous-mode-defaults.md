# Autonomous Mode Default Answers

`SKILL.md`의 **Autonomous Mode Override** 섹션에서 interview가 OFF일 때,
각 WHETHER 단계에 자동으로 채울 기본 템플릿.

## 사용법

1. 사용자 신호로 자율모드 감지 → 본 파일 참조
2. 각 단계 state 파일을 `[가정]/[결정]/[리서치]` 라벨로 자동 채움
3. 사용자 USER PROFILE / memory에서 컨텍스트 추출(투자자/취준생/1인회사 등)
4. 끝까지 실행 → DESIGN.md 한 번에 제출

## A-1 problem-frame.md 자동 채움

```markdown
# problem-frame

## 1. 한 문장 아이디어
[가장 일반적으로 사용자가 원하는 아이디어 — 한 문장 요약]

## 2. 겪는 사람
- **본인**: [USER PROFILE의 작업자 유형]
- **맥락**: [현재 활동 — 강의/취준/솔로 등]
- **규모**: 1인

## 3. 어떤 상황에서
- 자투리 시간에 [아이디어 카테고리] 떠오름
- 적을 곳/진행 방식 결정 못 함
- 며칠 후 잊어버림

## 4. 원인
- 위임 부재 (기획 행위를 본인이 함)
- 백로그 부재
- 방해 최소화 정책

## 5. 지금 치르는 비용
- **시간**: 주 N시간 (생성/검토/관리)
- **기회비용**: 다른 우선순위 시간
- **정신 비용**: 큰 그림 불안

## 6. 지금 어떻게 버티고 있나
- [USER PROFILE 도구 — 카톡/노션/메모장 등]

## Gate 판정
- 4요소 + 비용 구체 + 대안 1개+ → **GO**
```

## A-4 non-goals.md 자동 채움

3개 기본 비범위 (도메인 무관):

1. **자동 실행** — 외부 영향(Push/이슈/메시지)은 항상 수동 승인
2. **외부 수익화 도구** — 결제/CRM/B2B 연동 ❌ (자가소비 시스템)
3. **실시간 알림** — deliver: local, Discord 푸시 ❌ (방해 최소화)

추가: 도메인별 표준 비범위 (투자자 → 결제 ❌, 취준생 → 시간 ❌ 등)

## A-5 assumptions.md 자동 채움

5개 약한 가정 기본 세트:

1. `[가정]` 빈도 (매일 N건 = 적절)
2. `[가정]` 시간대 (평일 HH:MM KST)
3. `[결정]` 저장소 위치 (로컬 + 원격 2단계)
4. `[결정]` 트리거 듀얼 (자동 + 수동)
5. `[가정]` 단위수익 양수

각 가정에 **검증 시점** 명시 (2주 후 측정 등).

## 외부 영향 사전 동의 필요한 액션

자율모드에서도 다음은 사용자 명시적 동의 필수:

- **Push**: GitHub push, repo 생성/삭제, branch 작업
- **Payment**: Stripe/유료 결제/구독
- **Delete**: 파일 삭제, 외부 리소스 삭제
- **Issue 생성**: GitHub Issues, Linear 자동 생성
- **메시지 발송**: Discord DM, 외부 이메일/SMS

이 중 하나가 필요하면 사용자에게 한 줄로 확인 후 진행.
좌절 시그널("그만묻고") 시에도 외부 영향은 한 번은 묻는다.
